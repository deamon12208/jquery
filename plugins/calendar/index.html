Attach the calendar to one or more input fields with the following:

$('input selector').calendar();

or with initial settings:

$('input selector').calendar({autoPopUp: 'button'});
$('input selector').calendar({autoPopUp: 'both', 
	buttonImageOnly: true, buttonImage: 'calendar.gif', 
	buttonText: 'Calendar'});

The defaults are:

  • English
  • Date format dd/mm/yyyy
  • Month and year are selectable directly
  • Calendar starts on Sunday
  • Don't display days in other months
  • Day names are clickable to change the first day of the week
  • Clear/close controls show at the top
  • No date restrictions

Note that the available year range changes for each field. This is achieved with the following:

popUpCal.fieldSettings = customSettings;
		
function customSettings(input) {
	range = (input.className == 'calendarFocus' ? 5 : 
		(input.className == 'calendarButton' ? 7 : 10));
	return {yearRange: '-' + range + ':+' + range, 
		minDate: (input.id == 'dTo' ? getDate($('#dFrom')) : defaultMinDate), 
		maxDate: (input.id == 'dFrom' ? getDate($('#dTo')) : defaultMaxDate)};
}

The date range restrictions are also set in this custom function.

Restrict the dates as follows:

popUpCal.minDate = new Date(2005, 1 - 1, 26);
popUpCal.maxDate = new Date(2007, 1 - 1, 26);

Change the first day of the month as follows (0 = Sunday, 1 = Monday, ...):

popUpCal.firstDay = 1;
popUpCal.changeFirstDay = false;

Don't allow the month or year to be selected directly with:

popUpCal.changeMonth = false;
popUpCal.changeYear = false;

Change the date format as follows:

popUpCal.dateFormat = 'MDY/';

The first three characters indicate the day/month/year order, while the last is the separator character.

Change dates in other months as follows:

popUpCal.showOtherMonths = true;

Move the clear/close controls as follows:

popUpCal.closeAtTop = false;

Don't allow weekends to be selected as follows:

popUpCal.customDate = popUpCal.noWeekends;

Highlight (selected) national days as follows:

popUpCal.customDate = nationalDays;
natDays = [[1, 26, 'au'], [2, 6, 'nz'], [3, 17, 'ie'], [4, 27, 'za'], [5, 25, 'ar'], [6, 6, 'se'],
	[7, 4, 'us'], [8, 17, 'id'], [9, 7, 'br'], [10, 1, 'cn'], [11, 22, 'lb'], [12, 12, 'ke']];
function nationalDays(date) {
	for (i = 0; i < natDays.length; i++) {
		if (date.getMonth() == natDays[i][0] - 1 && date.getDate() == natDays[i][1]) {
			return [false, natDays[i][2] + '_day'];
		}
	}
	return [true, ''];
}

Use the jQuery Calendar

  1. Include the jQuery library in the head section of your page. 
    <script type="text/javascript" src="http://code.jquery.com/jquery-latest.pack.js"></script>
  2. Download and include the jQuery Calendar CSS and the jQuery Calendar Javascript in the head section of your page. 
    <style type="text/css">@import url(jquery-calendar.css);</style>
<script type="text/javascript" src="jquery-calendar.js"></script>
    Packed version (8KB) jquery-calendar.pack.js
  3. Configure calendar (if necessary) using the settings documentation. Here is an example: 
    popUpCal.autoPopUp = 'button';
  4. Initialize the calendar using the jQuery selector followed by the calendar function. 
    $('input selector').calendar();

jQuery Calendar Recent Updates

07/18/2007

  • Fixed IE select bug - needed to wait for show to complete before you got the size. Still doesn't do the borders quite right though.
  • Remove/disable prev/next links if not applicable - driven by hideIfNoPrevNext attribute.
  • Remove current link if not applicable.
  • Added showStandalone function that displays the calendar in a "popup dialog", complete with heading prompt and callback routine. If blockUI plugin is available it'll use that to make it modal.
    • See the Update button on the demo.
  • Moved setupActions method inline in HTML as this provided a render performance increase - I guess it didn't have to search for the controls to connect actions to, nor create new functions every time.

07/12/2007

  • Added basic support for an inline calendar. Set popUpCal.inline to true (NOTICE: This doesn't work with other calendar instances yet.)
  • Added support for a customEvent that has this.date as the selected date.
  • Fixed Bug where button caused the form to be submitted.
  • The 'change' event now fires when the input is filled in with a date. (Thanks SekYo)
  • Added Marc R's fix for when dateFormat has no separator. (example: "YMD")
  • Added exact height and width for calendar cover. (Thanks hjr)

07/01/2007

  • Added a new attribute, fieldSettings, which can be a function that takes an input field and returns a settings(anonymous) object to update the calendar with.
  • Added functionality that allows you to click on the day names to have the week start on that day. You can disable this feature by setting the changeFirstDay attribute to false.
  • Handling of disabling/enabling an input field and any trigger button/image - see enableFor() and disableFor().
  • Can make the month/year lists unavailable with changeMonth and changeYear.
  • Triggering function renamed to showFor(). This can be used externally (as one correspondent wanted - Ronald) by passing in the appropriate input field.
  • Removed getDayName() and getMonthName() as they served no real purpose.
  • Minor bugs fixes and tidy up.

06/17/2007

  • Calendar now displays full day names, so you gain more control over the display - names reduced accordingly (will need to update the Spanish version).
  • Added Clear link to close the calendar and erase the date value.
  • Added clearText property to set the text for the Clear link (will need to update the Spanish version).
  • Handle ctrl+end key to activate the Clear link.
  • Added buttonImageOnly property (true/false) to place an image after the field without a button.
  • Added closeAtTop property (true/false) to position the clear/close links at the top or bottom.
  • Added showOtherMonths property (true/false) to display dates in other months (non-selectable).
  • Changed autoPopUp property to be focus/button/both to allow for the combination.
  • Calendar now closes when clicking on the page outside of it.
  • Keyboard navigation continues to work after clicking a link.
  • Constructor function (calendar()) now accepts parameters to allow setting of attributes in one go (see demo for examples).
  • Custom date functionality added.
  • Bug fix SSL conection and iframe.
  • Updated demo page
    • to allow setting of opening speed
    • to demonstrate focus/button/both and buttonImageOnly properties
    • to show days in other months and clear/close at bottom as options
    • to show how constructor functions works with parameters
